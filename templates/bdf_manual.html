📘 BDCOM Devs Forum Manual

This manual helps you understand how to use and contribute to the BDCOM Devs Forum effectively.

📝 Creating a Post

  • Click the “Create” button (right corner of the Navigation bar).
  • Fill in the Title, Type (Query or Patch), Category, and the Content.
  • You can reference another post by selecting a Reference ID if needed.
  • Click “Submit” to publish.

🧩 What is Query & Patch?

  • Query is something that you have faced problematic or challenging. You may need help for this. Mainly it means - you are asking for help.
  • On the other hand Patch is a fancy name for Quick Fix or Solution. The solution can be either be:
    • In response to a specific Query (i.e., a previously raised problem or issue), or
    • A standalone tip, trick, workaround, or best practice that can help others avoid or resolve similar issues in the future.

💬 Giving Feedback

  • Open a post and scroll to the Feedback section.
  • Write your feedback and click “Give Feedback”.
  • If it’s your feedback, you can edit or delete it.

🌟 Endorsing Posts

  • Click the “🌟 Endorse” button under any post to show your support or agreement.
  • Each user can endorse a post only once. Clicking the button again will remove your endorsement.

📌 Add Reference to Post

  • You can optionally link your post to a previous Query or Patch by selecting it from the 🔗 Reference dropdown.
  • This helps others understand the context or origin of your post, especially when submitting a follow-up fix or related discussion.
  • Referencing posts promotes better traceability and keeps related topics connected.
  • ou can see whether a post has a reference attached by checking the post detail page—this information is displayed below the post content by Addressing to [post_title]

📬 Notices & Features

  • View current Notices in the right sidebar of the homepage.
  • See upcoming Features in the same section.
